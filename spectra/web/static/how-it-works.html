How Spectra Works

Spectra reads a forward-facing driving video frame by frame, builds a picture of the road and the traffic in it, and asks one question for every object: is this on a collision course with my lane? The map below is the full pipeline. Each stage shows the data it produces. Open any stage or the topic menu for the details.

Depth & Motion

Three separate cues judge whether an object is getting closer, then they are fused. Depth is the physical backbone. A per-track Kalman filter turns the metric distance into a smooth distance and closing speed, so the time to collision survives a constant scale error in the depth.

Depth (Metric, Kalman)

Takes the 25th-percentile distance from the box, then a Kalman filter smooths it.

Lane Analysis

Object positions are measured against the ego lane, where 0 is the lane center and 1 is the lane edge, so the same thresholds work across camera angles. Crossing risk is the chance an object will be in the ego lane at impact, and it gates the whole risk score.

Lane (UFLDv2)

Finds the ego lane corridor, smoothed by a Kalman filter that coasts on skipped frames.

Risk Score & State

The score is a single value from 0 to 1. Lane relevance is a multiplier, not an added term, so a close or fast object that is not in our path still scores near zero. The status is just a banding of that score, with one escalation for an imminent collision.

signal    = 0.40·eta_pressure + 0.30·proximity + 0.25·approach + 0.05·brake
gate      = 0.65 + 0.35·confidence
relevance = crossing_risk
risk.score = gate · relevance · signal

State

0.60 and up is DANGER, 0.25 and up is CAUTION, otherwise SAFE. A confirmed ttc under 1s on an already-CAUTION object jumps to DANGER. Detection confidence under 0.20 forces SAFE. Hysteresis then smooths flicker, 3 frames up and 7 down.
