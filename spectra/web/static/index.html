Overall Status

Collision ETA
Distance ÷ closing speed, both from the depth Kalman (metric depth — not visual cues). Shows a value only while the object is actually closing; otherwise Estimating / No closing / Low confidence.
 —
Risk Score / 100
How dangerous the object is, 0–100. See "What builds the Risk Score" below for the weighted signals and multipliers.
 —

What builds the Collision ETA
ETA = Distance ÷ Closing speed — both from the depth Kalman. It is purely longitudinal; lateral position (lane) feeds the Risk Score, not the ETA.

Distance
Metric distance to the object from monocular depth (Depth Anything V2 Metric). Bar gauges closeness over 0–60 m.
—
Closing speed
How fast the gap is closing, from the per-track depth Kalman (range-rate). Bar gauges speed over 0–12 m/s.
—

What builds the Risk Score
Weighted signals add up, then are multiplied by lane relevance and confidence.

ETA pressure 40%
How close the collision ETA is to the 3s caution horizon. Estimated metric distance and closing speed drive it.
—
Proximity 30%
How near the object is, derived from the metric depth distance (Depth Anything V2 Metric).
—
Approach 25%
Relative closing speed, indicating how fast the object is nearing the camera.
—
Brake 5%
Brake-light cue for a lead vehicle — an early warning before the gap visibly closes.
—
Lane relevance
Multiplier: how likely the object is in your lane at impact. Off-path objects gate the whole score toward zero.
—
Confidence
Multiplier: detection / tracking / depth reliability.

How Spectra Works

          

        

        
      


      

        

          

            
            

              

                
                
01

              

              
Data Ingestion

              
Frames are streamed through a WebSocket for real-time preview while the backend initializes the vision
                stack.

              

                FastAPI
                WebSockets
              

            


            
            

              

                
                
02

              

              
Road Geometry

              
Runs UFLDv2 every 3rd frame by default, reuses the smoothed ego-lane corridor between updates, and
                normalizes object positions relative to that lane.

              

                Lane ROI
                Vanishing Point
              

            


            
            

              

                
                
03

              

              
Depth (Spatial)

              
Depth Anything V2 Metric VKITTI estimates object distance in meters. A normalized nearness map is
                derived only for compatibility with filtering and scoring.

              

                Metric Depth
                80m Range
              

            


            
            

              

                
                
04

              

              
Motion Analysis

              
DIS Optical Flow runs every frame by default. RANSAC-based homography compensates for camera
                ego-motion before object-level motion is scored.

              

                OpenCV DIS
                Ego-Motion
              

            


            
            

              

                
                
05

              

              
Object Tracking

              
YOLO detects road participants every 3rd frame by default. The tracker propagates objects between
                detections while display IDs are numbered per class, such as Car #1 and Truck #1.

              

                YOLOv8
                IoU Tracker
              

            


            
            

              

                
                
06

              

              
Risk Fusion

              
Separates physical Collision ETA from risk evidence. Depth + Kalman estimates range motion, while
                bbox growth, optical flow, lane crossing and brake cues explain the risk evaluation.
              

              

                Collision ETA
                Risk Evidence
              

            


            
            

              

                
                
07

              

              
Decision Engine

              
Classifies state as SAFE, CAUTION, or DANGER. Hysteresis stabilization prevents jitter in risk
                transitions.

              

                Hysteresis
                State Machine
              

            


            
            

              

                
                
08

              

              
Efficiency

              
SAFE objects remain tracked for continuity but are hidden from the object list and overlay so the view
                stays focused on CAUTION and DANGER cases.

              

                Telemetry
                Deferred Render
